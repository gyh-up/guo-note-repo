## 时间复杂度

### 介绍

- **只关注循环执行次数最多的一段代码**
- **总复杂度等于最高阶项的复杂度**
- **嵌套代码的复杂度等于嵌套内外代码复杂度的乘积**
- **固定常量执行次数对执行次数的增长趋势无影响**

### **推导大O阶**

1. **用常数 1 取代运行时间中的所有加法常数**
2. **在修改后的运行次数函数中，只保留最高阶项**
3. **如果最高阶项存在且不是 1，则去除与这个项相乘的常数。**
4. **得到的结果就是大O阶。**

#### 例 1：

```go
// 执行了 1 次
sum := 0
// 循环执行了n次
for i := 0; i < n; i++{
    sum += i
}
// 执行了 1 次
return sum
```

上述时间复杂度为 O(n)

#### 例 2：

```go
// 执行了 1 次
sum := 0
// 循环执行了 100 次，对增长趋势无影响
for i := 0; i < 100; i++{
    sum += i
}
// 循环执行了 n 次
for i := 0; i < n; i++{
    // 循环执行了 n 次
    for j := 0; j < n; i++{
        sum += i
    }
}
// 执行了 1 次
return sum
```

上述时间复杂度为 O(n^2)

#### 例 3 ：

```go
i := 1
for i <= n {
	i = i * 2
}
```

上述时间复杂度为 O(logn)
